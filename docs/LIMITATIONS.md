# Limitations and claim boundaries

- Hedge supports a narrow TypeScript target: Next.js App Router, Next.js Pages API routes, basic Express, common Prisma, object-storage, network, workflow, and credential patterns.
- Handler-scoped AST analysis is not complete interprocedural data flow. Same-file helper propagation and supported middleware are modeled, but imported helpers, decorators, generated routes, runtime-computed matchers, infrastructure policy, and deployment topology may remain unknown.
- Source-only smoke tests on one App Router, one Pages API, and one Express repository confirmed framework detection, benign-change silence, and exact evidence for a supported storage change. The Next.js repositories also demonstrated the expected partial-coverage warnings for a complex middleware matcher and an unresolved imported authentication helper; these three tests are compatibility evidence, not a representative accuracy sample.
- A detected control means relevant code is associated with the supported handler or middleware path; it does not prove semantic correctness, runtime reachability, or complete enforcement.
- Working-tree collection refuses symlinks and out-of-root real paths. PR comparison instead reads bounded regular blobs directly from exact Git commits; the required base and head objects must exist in the local object database.
- A surfaced risk is not an exploitability verdict.
- CWE and STRIDE labels are optional metadata, not the core evidence.
- The competition workflow is same-repository only. Its `pull_request_target` definition is loaded from the trusted base branch, every job repeats the same-repository guard, and the exact head is handled only as data in the secretless collector. Fork-safe operation and any target-code execution need a separate approval or credential architecture.
- The v0.5.2 candidate passed public CI and its Action self-test, and the judge-lab canary completed credential-separated collection, GPT-5.6 reasoning, publication, exact evidence, and a recorded `BLOCK` decision. The final release tag and release-bundle smoke test still belong to the release step.
- Counterfactual verification depends on a reviewed, repository-owned self-contained witness and legitimate-behavior script. Hedge seals the witness bytes, runs both revisions and legitimate behavior without network or credentials, and derives the exact graph change automatically; these controls do not make a poorly designed witness semantically authoritative.
- The frozen API-backed batch measured routing, exact-evidence validity, stability, token usage, latency, failures, and the instruction boundary on ten synthetic cases. It is not a precision, cost, general-accuracy, or vulnerability-detection measurement.
- Model spend is bounded by routing, serialized-request bytes, patch bytes, output tokens, reasoning effort, and retry controls, but these limits do not predict a dollar amount or establish an exact input-token count. Luna allows at most 384 output tokens with minimal reasoning, a 12 KiB patch payload, and a 48 KiB serialized request; Sol allows at most 4,096 output tokens with low reasoning, a 48 KiB patch payload, and a 160 KiB serialized request. Output-token limits include visible and reasoning tokens, and every accepted claim must still resolve to exact evidence. A failed provider request may not return usage, so reported token totals cover provider-reported completed calls rather than guessing missing spend.
- Hedge does not issue explicit prompt-cache writes. Provider-default implicit caching may apply, but an explicit cache policy is deferred until measured stable-prefix reuse shows that cache-write overhead is worthwhile.
- The isolated live harness recorded all 30 runs against the separate frozen ten-pair corpus, but 12 model-routed runs failed and the operational gate is `FAIL`. Accepted results retained 100% exact-evidence validity and the instruction boundary held; exact signatures were stable for only two cases. The bounded human adjudication was completed on 2026-07-20 without changing the frozen corpus or generated results, and the corpus remains a small synthetic sample rather than the deferred expert or adversarial corpus.
- The staged publisher's complete collect/reason/publish path is proven by one live same-repository judge-lab canary with a real credential. That single run does not establish fleet reliability or support fork pull requests.
- The approval-gated Codex job produced a source-bound, digest-bound three-file patch, but the target fixture's generic `npm test` command had no Vitest files and the automated publisher correctly did not open a PR. The exact artifact was recovered into an explicitly experimental draft after its dedicated witness and legitimate-behavior commands passed without credentials. This proves bounded patch production, not a fully successful automated remediation contract.
- Reviewable model-refresh, risk-acceptance, remediation, and verification-state publication requires the target repository to allow GitHub Actions to create pull requests. Hedge gives those publisher jobs explicit `contents: write` and `pull-requests: write` permissions, but GitHub's repository-level setting can still deny PR creation; the workflows fail instead of writing directly to the protected branch.
- One hardened remote verification run passed all four requirements and recorded HEDGE-009 as `verified` through a reviewable state PR. The workflow refuses to record state unless `state_ref` is the current protected default branch, rechecks that branch's exact SHA before publication, and pins target execution to the reviewed Node 22 container digest. One canary does not establish witness quality or verification reliability across arbitrary repositories.
- The public dashboard is a recorded deterministic artifact for no-install judging; it does not claim to be the output of a live GitHub or GPT-5.6 run.
- The current risk ID allocator is stable against the committed register; brand-new uncommitted findings can be renumbered if the set/order of simultaneous new findings changes before merge. Semantic graph identities are line-independent, but structural refactors can still legitimately alter them.
- Hedge does not replace SAST, DAST, dependency scanning, secrets scanning, penetration testing, architecture review, or human judgment.
- The deadline build deliberately defers the full TypeScript Program, complete monorepo analysis, additional framework adapters, fork-PR broker, v1 configuration migration, hosted fleet views, Windows validation, and the large held-out expert corpus. These remain tracked in `docs/PRODUCT_MASTER_PLAN.md` rather than being implied by current behavior.
