# Submission checklist

## Challenge requirements supplied in the brief

- [ ] Working project built with Codex using GPT-5.6.
- [ ] Category selected: Developer Tools.
- [ ] Project description explains what was built and how it works.
- [ ] Public YouTube demo is under three minutes.
- [ ] Demo audio explicitly explains how both Codex and GPT-5.6 were used.
- [ ] Repository is public with relevant licensing, or private and shared with `testing@devpost.com` and `build-week-event@openai.com`.
- [ ] README includes setup instructions, sample data, and clear run guidance.
- [ ] Submission highlights where Codex accelerated the workflow and where key decisions were made.
- [ ] `/feedback` Codex Session ID is from the session where the majority of core functionality was built.

## Project evidence

- [ ] All competition work has dated commits inside the allowed window.
- [ ] Baseline tag or branch records any pre-existing foundation.
- [ ] `DECISIONS.md` distinguishes human and model decisions.
- [ ] Eval results include raw counts and limitations.
- [ ] Exact model IDs and evaluation date recorded.
- [ ] Cost claim is measured, not guessed.
- [ ] Fresh clone succeeds from README instructions.
- [ ] Demo branches are prepared and outputs verified.
- [ ] No secret appears in recordings, logs, fixtures, commits, or screenshots.

## Final language review

- [ ] “Surfaces risks,” not “finds vulnerabilities.”
- [ ] “Suggested witness,” not “proof,” until executed.
- [ ] No claim of universal language/framework support.
- [ ] No claim that every risk can be automatically repaired.
- [ ] Limitations are visible and concise.
